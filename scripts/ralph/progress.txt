## Codebase Patterns
- Tests must run inside Docker: `docker compose exec app php artisan test` (DB is only available inside the container)
- Config keys in `config/grmrl.php` map from `.env` vars; always add corresponding `.env.example` entries when adding new config keys
- Layout at `resources/views/layouts/app.blade.php` uses `config('grmrl.*')` for dynamic values in header
- `config('grmrl.support_phone')` e `config('grmrl.support_email')` sono i contatti di supporto usati in header e footer; `config('grmrl.phone')` e `config('grmrl.email')` sono i contatti CAI usati nelle email transazionali
- Il form di registrazione usa Alpine.js per la gestione dinamica dei minori (array) e per l'autocomplete delle sezioni CAI; i dati Alpine vanno inizializzati con `old()` passato come JSON per preservarli dopo errori di validazione
- L'endpoint `/api/sections?q=...` restituisce le sezioni CAI per l'autocomplete; il rate limiting è configurato in RouteServiceProvider
