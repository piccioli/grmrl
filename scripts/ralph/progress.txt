## Codebase Patterns
- Artisan commands must be run via `docker compose exec app php artisan ...` (DB is in Docker, not accessible from host)
- Tests use `php artisan test` (no PHPStan, just Pint and Pest/PHPUnit)
- Migrations use `2026_06_25_XXXXXX_` prefix; add columns with `->after()` for clean ordering
- Use `Illuminate\Support\Facades\Http` for external HTTP calls (Laravel Http client is available); Pint enforces `concat_space` so use `' '` not `" "` for string concat
- Always run `pint` (without `--test`) to auto-fix style before committing; there may be pre-existing issues in unrelated files that Pint will fix as part of the commit
- RegistrationController maps Activity model fields to an array for the view; add new model fields there when they need to be accessible in Blade templates
- Filament v3 `Placeholder::make()->content(fn (?Model $record) => ...)` reliably injects `$record`; use `HtmlString(view(...)->render())` to embed custom Blade components with Alpine.js inside Filament forms
- In Blade templates, `$refs` inside Alpine.js `x-data` attributes is NOT processed by PHP (only `{{ $var }}` is); safe to use Alpine.js `this.$refs.x` in x-data strings
